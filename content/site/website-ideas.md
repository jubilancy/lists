# future features of my website
- Build a starter Astro Starlight documentation site configured for GitHub Pages deployment with sample Markdown pages, search, and light/dark mode switch. Here is the succinct list of features for your digital garden wiki, optimized for your query:  
- Sticky Top Navigation: Fixed header with primary categories and a mobile hamburger menu for a "curated portal" feel.  
- Mega-Menu Dropdowns: Hover-based menus in the top nav to reveal site architecture without a sidebar.  
- Automated Sitemap: A dynamic sitemap.xml and /sitemap page that auto-rebuilds via GitHub Actions upon every .md edit.  
- Omnisearch (Cmd+K): High-speed fuzzy search that indexes both page titles and specific list items (vital for your dictionary).  
- Breadcrumb Trails: Persistent navigation paths (e.g., Home > Category > Page) below the header for constant context.  
- Bi-directional Backlinks: A "linked mentions" footer surfacing all organic connections to the current note.  
- Hover Previews: Pop-up snippets for internal links to allow non-linear browsing without page hops.  
- Maturity Tags: Visual status icons (🌱 Seedling, 🌿 Budding, 🌳 Evergreen) to signal content readiness. 
- Graph View Overlay: A toggleable, interactive map visualizing the interconnections of your notes.  
- Themed Callouts: Styled info blocks using your specific palette (#f4a261 and #2a9d8f).  
- Auto-generated Indexes: Dynamic A–Z and "Recent Additions" pages to eliminate manual directory updates.  
- Last Tended Dates: Timestamps automatically pulled from Git commit history to show note freshness.  
- Subdomain Integration: A global header linking dictionary.eliana.lol back to the eliana.lol root.  
- In-Browser Editing: "Edit this Page" links to the GitHub Web Editor for a "click-and-save" workflow.  
- Dark Mode Toggle: A primary switch in the top nav for accessibility.  
- On-Page TOC: A minimalist right-side menu for jumping between headers in long-form notes.
